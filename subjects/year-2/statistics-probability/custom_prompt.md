## طبيعة المادة

| النوع | الاستخدام | أمثلة |
| --- | --- | --- |
| نظري | شرح المفاهيم الإحصائية والاحتمالية | `Central Tendency`، `Probability Distributions`، `Hypothesis Testing` |

## 🎯 أنواع أسئلة إضافية إلزامية للأتمتة (الفحص بالكامل آلي)

> هذه الأنواع تُدمَج **ضمن** الأجزاء أعلاه (لا تُنشئ لها أقساماً ## منفصلة) — التزم بها لضمان كفاية وتنوّع الأسئلة القابلة للتصحيح الآلي.

### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / theory (≥15، easy)
**Q{N}:** سؤال مراجعة مباشر (تعريف، فرق، متى نستخدم). **A:** إجابة مختصرة قابلة للمقارنة الآلية.
مثال: «ما الفرق بين البيانات الكمية والوصفية؟»

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / theory (≥3، hard)
### السيناريو {N}: {عنوان}
> [وصف الحالة: جدول تكراري، مجموعة بيانات، أو مسألة احتمالية كاملة — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو**.
المصادر: `numerical_case`، `algorithm_state`، `data_analysis`.
مثال: «جدول تكراري لبيانات الهيموغلوبين — أي فئة لها أعلى تكرار نسبي؟»

### تطبيق عملي (أكمل / صحّح / نفّذ) — ضمن exercise / analysis_exercise (≥5، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**. الإجابة قابلة للمقارنة رقمياً.
الأنواع: `fill_gaps`، `code_fix`، `table_fill`، `numerical_solve`، `diagram_completion`.

### تتبع خطوة بخطوة (جدول ناقص) — ضمن trace_exercise / mcq (≥4، medium)
**المدخل:** {حالة ابتدائية} → **أكمل الجدول** → **نموذج الحل**.
الأهداف: `algorithm_iterations`، `variable_state`، `search_tree_expansion`.

### مقارنة وتحليل — ضمن mcq / theory / analysis_exercise (≥5، medium)
سؤال مقارنة بخيارات أو جدول: | المعيار | الخيار A | الخيار B |
مثال: «متى يُفضَّل استخدام الوسيط `Median` على الوسط الحسابي `Mean`؟»

### اكتشف الخطأ (حساب / منطق) — ضمن exercise (≥4، hard)
**الحساب/الجدول (يحتوي خطأ):** … **اكتشف الخطأ:** … **التصحيح:** … + شرح الحل (3 نقاط).
أنواع الأخطاء: `misconception`، `off_by_one`، `wrong_formula`، `calculation_error`.

### نظري منظم — ضمن theory / mcq (≥10، medium)
### السؤال {N}: {السؤال}
**نموذج الإجابة:** 1. التعريف 2. المكونات/الشروط 3. مثال 4. متى نستخدم.

## قواعد الكتل داخل الشرح

**📊 المخطط** — قسمان بالترتيب: ما هذا المخطط؟ + جدول وصف العناصر/الفئات + بلوك `diagram`.
النوع `type:` يجب أن يكون فقط من: `histogram`، `frequency-polygon`، `pie-chart`، `stem-and-leaf`.
**ملاحظة:** هذه مخططات إحصائية لعرض بيانات، **لا حاجة لجدول الروابط** (لا يوجد عُقد/أسهم منطقية بينها) — اكتفِ بجدول العناصر/الفئات فقط. انظر SCHEMA.md v1.0 §Diagram.

**⚙️ الخطوات / الخوارزمية** — أسطر داخل fence ` ```algorithm ` بصيغة `1 | الخطوة | الأداة | ماذا يحدث` (سطر لكل خطوة). لا ترسم مربعات ولا أسهم — البارسر يحوّلها. استخدمه لكل إجراء حسابي مرتّب: حساب `Standard Deviation`، بناء `Frequency Table`، خطوات اختبار فرضية، خوارزمية `Bayes' Theorem`. انظر SCHEMA.md v1.0 §Algorithm.

**الفهم الخاطئ ❌** / **الفهم الصحيح ✅** — سطر واحد لكل منهما.

#### مهم للامتحان ⚠️ / #### نقطة مهمة ⚠️ / #### ملاحظة: / #### الدرس المستفاد:
← `blockquote` يتبع كل منها مباشرةً.

**🤔 تفعيل الفهم** — استخدمه **≥3 مرات** في كل محاضرة.

حساب أو صيغة ناقصة — ضع `_______` مكان القيمة أو الرمز الناقص (وليس كوداً برمجياً، فالمادة نظرية بحتة).

تصحيح حساب أو صيغة خاطئة — قدّم الحساب الخاطئ أولاً ثم الحساب الصحيح + شرح مصدر الخطأ.

**💡 التشبيه** — بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md v1.0 §Analogy.

**⚖️ المقايضة** — جدول: المزايا | العيوب | متى تختاره. مثال: `Mean` مقابل `Median` كمقياس نزعة مركزية. انظر SCHEMA.md v1.0 §Trade-off.

**🔄 قبل / بعد** — حالة البيانات قبل عملية إحصائية (تجميع، تطبيع، ترتيب) وبعدها + جملة «ماذا تغيّر؟». انظر SCHEMA.md v1.0 §Before-After.

**🔍 تتبع التنفيذ** (inline) — المدخل (بيانات خام) + جدول الخطوات (مثل: الخطوة | العملية الحسابية | القيمة الناتجة) + النتيجة النهائية. خصّص الأعمدة حسب الموضوع (جدول احتمالات تراكمية، خطوات بناء توزيع). انظر SCHEMA.md v1.0 §Trace.

**📐 المعادلة** — LaTeX داخل `$$` أو fence `math`. يتبعها **الشرح:** بمعنى كل رمز (`X̄`، `Me`، `Mo`، `σ`، `σ²`، `Q1`، `Q3`، `P(A)`، `E(X)`). لا تخلط أكثر من معادلة غير مرتبطة في نفس البلوك. انظر SCHEMA.md v1.0 §Equation.
